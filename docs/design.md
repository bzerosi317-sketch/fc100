## 1. 系统定位与架构

### 1.1 系统定位

公网无人机管理系统是部署在**阿里云公网服务器**上的无人机统一管控平台。核心职责：

- **向下**：对接大疆司运平台（FC100）、大疆上云API（御3/Pilot 2）、Skyway2 私有版（机场），统一接入多型号无人机
- **向上**：通过物联管理平台和统一视频平台，将设备数据和视频流同步到应急指挥系统（ECS）
- **自身**：提供 Web UI 用于设备拓扑、任务执行、实时监控

### 1.2 接入设备

| 设备 | 型号 | 接入平台 |
|------|------|---------|
| 大载重无人机 | 大疆 FC100 | 大疆司运平台（运载云端互联） |
| 单兵多旋翼无人机 | 大疆御3 行业版 | 上云API（Pilot 2 上云） |
| 便携式机场 | 大疆 M4TD | Skyway2 私有版（边缘节点桥接） |
| 固定翼无人机 | 纵横 CW-15 | 厂家地面站系统（下一期） |

## 2. 用户登录模块

### 2.1 设计原则

- 系统所有页面（除登录页）均需登录后访问，未登录自动重定向至 `/login`
- 登录凭证通过 **系统设置 → 人员管理页面** 中创建的人员账号和密码进行登录
- 使用 JWT Token 进行身份认证，支持 access_token + refresh_token 双令牌机制
- 退出登录清除 Token，重定向至登录页

### 2.2 登录页面设计

```
┌─ 公网无人机管理系统 ────────────────────────────────────────┐
│                                                               │
│                     ┌─────────────────┐                       │
│                     │    🛸 系统 LOGO  │                       │
│                     │  公网无人机管理系统 │                       │
│                     │                 │                       │
│                     │  人员账号:       │                       │
│                     │  [______________] │                       │
│                     │                 │                       │
│                     │  密码:           │                       │
│                     │  [______________] │                       │
│                     │                 │                       │
│                     │  [登  录]        │                       │
│                     │                 │                       │
│                     │  ────────────── │                       │
│                     │  登录失败提示信息   │                       │
│                     └─────────────────┘                       │
│                                                               │
└───────────────────────────────────────────────────────────────┘
```

### 2.3 认证流程

1. 用户访问系统，未检测到有效 Token → 自动跳转至 `/login`
2. 输入人员账号和密码，点击「登录」
3. 前端调用登录接口提交凭证
4. 后端验证账号密码 → 返回 JWT 令牌（access_token + refresh_token）
5. 前端将 Token 存入 localStorage，后续请求在 `Authorization` 头部携带 `Bearer access_token`
6. 页面跳转至 `/dashboard` 数据看板
7. Token 过期后前端自动使用 refresh_token 刷新，刷新失败则跳回登录页
8. 用户点击「退出登录」→ 清除本地 Token → 重定向至 `/login`

### 4.3 字段说明

**1. 计划时间 | 实际时间**
- **计划时间**：任务预估执行时间（创建时设定）
- **实际时间**：飞行任务真实的执行起止时间；未执行显示 —

**2. 执行状态**
- **未完成**：待执行、执行中、已暂停、已挂起
- **已完成**：部分执行、执行成功、执行失败、执行终止

**3. 类型**
- **勘察任务**：用于线路巡视、勘察的任务
- **运载任务**：大载重无人机执行的运载任务

**4. 计划名称**：显示用户创建任务时定义的名称。系统默认生成规则：`新建计划_时间戳`（例如：`新建计划_202605151430`）

**5. 航线名称**：显示该飞行任务执行时使用的航线副本（只读快照）

**6. 设备名称**：显示参与该任务的设备名称

**7. 创建人**：显示创建该任务的用户信息

**8. 媒体上传**：显示飞行器在任务过程中拍摄的媒体文件上传状态：

| 状态 | 说明 |
|------|------|
| 待上传 | 媒体文件尚未开始上传 |
| 上传中 | 正在上传，显示 `已上传数量/总数量` |
| 上传结束 | 所有媒体文件上传完成 |

**9. 操作**
- 待执行任务：支持编辑、挂起、删除
- 已挂起任务：支持取消挂起（恢复待执行）
- **运载任务**：待执行状态时显示「开始任务」按钮，手动触发任务执行

### 4.4 创建任务页面（向导）

## 6. 实时画面模块

### 6.1 定位

以视频流为主体，将设备实时数据、任务进度、控制指令以面板形式附着于视频周围。页面分为两级：

| 多路画面 (/live) | 驾驶舱 (/live/:deviceId/cockpit) |
|-----------------|-----------------------------------|
| 同时查看多台设备的实时视频 | 专注单台设备，视频占据主导 |
| 通用监控/巡检用途 | 设备监控 + 任务全流程跟踪 |
| 设备卡片列表展示 | 右侧面板按 Tab 切换（仪表盘/任务/告警） |
| 可看任何在线设备 | 控制指令以浮动覆盖层呈现于视频之上 |

**交互流程**：多路画面 → 点击设备卡片「驾驶舱」按钮 → 进入该设备的驾驶舱 → 在驾驶舱内查看仪表盘、操控设备、跟踪任务 → 点击「返回多路」回到多路画面。

### 10.3 流媒体配置

## 11. 告警机制

### 11.1 设计原则

不设独立告警模块，告警直接嵌入驾驶舱页面。

### 11.2 告警展示位置

| 位置 | 展示方式 | 适用页面 |
|------|---------|---------|
| **驾驶舱底部面板 Tab** | 底部面板「🔔 告警」Tab 内按级别分组 | 驾驶舱 |
| **全局铃铛图标** | 页面右上角常驻，红色数字标记未读数 | 所有页面 |

### 11.3 告警级别

| 级别 | 颜色 | 示例 |
|------|------|------|
| CRITICAL | 红色 | 设备离线、起落架异常、电机故障、绞车异常 |
| WARNING | 黄色 | 电量低于20%、GPS弱、任务执行异常 |
| INFO | 蓝色 | 任务完成、设备上线 |

### 11.4 告警类型

| 告警类型 | 级别 | 说明 |
|---------|------|------|
| DEVICE_OFFLINE | CRITICAL | 设备离线 |
| BATTERY_LOW | WARNING | 电量低 |
| GPS_WEAK | INFO | GPS信号弱 |
| MOTOR_FAILURE | CRITICAL | 电机故障 |
| HOIST_ERROR | CRITICAL | 绞车异常 |
| TASK_ERROR | WARNING | 任务执行异常 |
