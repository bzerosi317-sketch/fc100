## 1. 系统定位与架构

### 1.1 系统定位

公网无人机管理系统是部署在**阿里云公网服务器**上的无人机统一管控平台。核心职责：

- **向下**：对接大疆司运平台（FC100）、大疆上云API（御3/Pilot 2）、Skyway2 私有版（机场），统一接入多型号无人机
- **向上**：通过物联管理平台和统一视频平台，将设备数据和视频流同步到应急指挥系统（ECS）
- **自身**：提供 Web UI 用于设备拓扑、任务执行、实时监控

### 1.2 接入设备

| 设备 | 型号 | 接入平台 |
|------|------|---------|
| 大载重无人机 | 大疆 FC100 | 大疆司运平台（运载云端互联） |
| 单兵多旋翼无人机 | 大疆御3 行业版 | 上云API（Pilot 2 上云） |
| 便携式机场 | 大疆 M4TD | Skyway2 私有版（边缘节点桥接） |
| 固定翼无人机 | 纵横 CW-15 | 厂家地面站系统（下一期） |

## 2. 用户登录模块

### 2.1 设计原则

- 系统所有页面（除登录页）均需登录后访问，未登录自动重定向至 `/login`
- 登录凭证通过 **系统设置 → 人员管理页面** 中创建的人员账号和密码进行登录
- 使用 JWT Token 进行身份认证，支持 access_token + refresh_token 双令牌机制
- 退出登录清除 Token，重定向至登录页

### 2.2 页面路由

| 路由 | 页面 | 说明 |
|------|------|------|
| `/login` | 用户登录 | 未登录时自动重定向至此页 |
| — | — | 登录成功后跳转至 `/dashboard` |

### 2.3 登录页面设计

```

│                     ┌─────────────────┐                       │

│                     │  公网无人机管理系统 │                       │
│                     │                 │                       │
│                     │  人员账号:       │                       │
│                     │  [______________] │                       │
│                     │                 │                       │

```

### 2.4 认证流程

1. 用户访问系统，未检测到有效 Token → 自动跳转至 `/login`
2. 输入人员账号和密码，点击「登录」
3. 前端调用 `POST /api/v1/auth/login` 提交凭证
4. 后端验证账号密码 → 返回 JWT 令牌（access_token + refresh_token）
5. 前端将 Token 存入 localStorage，后续请求在 `Authorization` 头部携带 `Bearer access_token`
6. 页面跳转至 `/dashboard` 数据看板
7. Token 过期后前端自动使用 refresh_token 刷新，刷新失败则跳回登录页
8. 用户点击「退出登录」→ 清除本地 Token → 重定向至 `/login`

### 2.5 密码管理

- 人员账号在系统设置 → 人员管理页面中创建时需设置初始密码
- 已创建的人员可修改密码：`PUT /api/v1/users/{id}/password`
- 密码加密存储（bcrypt），不支持明文找回

### 2.6 认证 API

| 方法 | 路径 | 说明 | 角色 |
|------|------|------|------|
| POST | `/api/v1/auth/login` | 用户登录 | 公开 |
| POST | `/api/v1/auth/refresh` | 刷新 Token | 登录用户 |
| POST | `/api/v1/auth/logout` | 退出登录 | 登录用户 |
| GET | `/api/v1/auth/me` | 获取当前用户信息 | 登录用户 |

### 3.4 区域说明

#### 3.4.1 统计卡片行

| 指标 | 数据来源 | 更新频率 | 点击行为 |
|------|---------|---------|---------|
| 设备总数 | `GET /api/v1/devices` 计数 | 每 10s | 跳转 `/devices` |

- 数据来源：`GET /api/v1/devices`

#### 3.4.3 任务状态

- 横向柱状图展示任务状态分布：执行中 / 已完成 / 失败 / 待执行
- 数据来源：`GET /api/v1/missions`

#### 3.4.4 在线设备快捷入口

- 展示当前在线设备卡片列表，每张卡片包含设备名称、型号、在线状态

- 点击卡片跳转至 `/live` 多路画面
- 数据来源：`GET /api/v1/live/streams`

#### 3.4.5 最近任务

- 展示最近 5-8 条任务记录
- 显示：计划时间、执行状态（带颜色标识）、任务名称、航线名称、设备名称、媒体上传进度
- 点击 [查看全部任务] 跳转 `/missions`
- 数据来源：`GET /api/v1/missions?page=1&size=8`

### 3.5 数据聚合 API

| 方法 | 路径 | 说明 |

| GET | `/api/v1/dashboard/stats` | 统计卡片数据（设备/任务计数） |

#### 字段说明

**1. 计划时间 | 实际时间**
- **计划时间**：任务预估执行时间（创建时设定）
- **实际时间**：飞行任务真实的执行起止时间；未执行显示 —

**2. 执行状态**
- **未完成**：待执行、执行中、已暂停、已挂起
- **已完成**：部分执行、执行成功、执行失败、执行终止

**3. 类型**
- **勘察任务**：用于线路巡视，勘察的任务
- **运载任务**：大载重无人机执行的运载任务

**4. 计划名称**：显示用户创建任务时定义的名称。

**5. 航线名称**：显示该飞行任务执行时使用的航线副本。点击航线名称可查看航线详情。航线副本为只读快照，不支持编辑和保存。

**6. 设备名称**：显示参与该任务的设备名称。

**7. 创建人**：显示创建该任务的用户信息。

**8. 媒体上传**：显示飞行器在任务过程中拍摄的媒体文件上传状态和进度：

| 状态 | 说明 |
|------|------|
| 待上传 | 媒体文件尚未开始上传 |
| 上传中 | 正在上传，显示 `已上传数量/总数量` |
| 上传结束 | 所有媒体文件上传完成 |

- **上传顺序**：优先上传即时任务拍摄的媒体文件；单次定时和重复定时任务按实际完成时间依次上传
- **立即上传**：点击待上传媒体文件的 ⬆️ 图标，立即上传该媒体文件
- **跳转媒体库**：点击上传进度区域，跳转至媒体库查看对应媒体文件
- 仅支持展示自动任务的媒体上传信息

**9. 操作**
- 待执行任务：支持编辑、挂起、删除
- 已挂起任务：支持取消挂起（恢复待执行）
- 飞行任务超过执行时间后，自动提示执行失败

```
     → 完成 (COMPLETED) ✅
     → 中断 (FAILED) ❌
     → 取消 (CANCELLED) ⛔
```

## 6. 实时画面模块

### 6.1 定位

以视频流为主体，将设备实时数据、任务进度、控制指令以面板形式附着于视频周围。页面分为两级：

| 多路画面 (/live) | 驾驶舱 (/live/:deviceId/cockpit) |
|-----------------|-----------------------------------|
| 同时查看多台设备的实时视频 | 专注单台设备，视频占据主导 |
| 通用监控/巡检用途 | 设备监控 + 任务全流程跟踪 |
| 设备卡片选择栏浓缩 OSD 数据 | 右侧面板按 Tab 切换（仪表盘/任务/告警） |

- 支持单条新建/编辑
- 支持批量导入（.xlsx / .csv），自动匹配线路和杆塔编号

### 8.5 航点-杆塔关联

在航线预览弹窗（5.5）中，支持将航点与杆塔绑定：

- 点击航点标记 → 显示「关联杆塔」下拉框，选择已有杆塔或新建杆塔
- 关联后该航点图标变为 🏗️ 标记
- 任务执行时，该航点拍摄的照片自动标记关联杆塔 ID
- 媒体中心筛选条件增加「杆塔」维度，可按指定杆塔查看全部巡检照片

### 13.11 设备型号映射

| 平台 | 原始型号 | 我们的 DeviceType |
|------|---------|-----------------|
| 司运 | 0-78-0 (FC100飞机) | FC100 |
| 司运 | 0-80-0 (FC30飞机) | FC30 |
| 司运 | 2-119-0 (FC100遥控器) | RC（遥控器，关联不独立注册） |

| 上云API | 0-71-0 (M3E/M3T) | MAVIC_3 |
| Skyway2 | 机场类型标识 | DOCK_M4TD |

### 13.12 设备生命周期状态机

```
UNKNOWN → (收到 bind/topo/首次 osd) → REGISTERED
REGISTERED → (topo online=true) → ONLINE
ONLINE → (topo online=false / 60s 超时) → OFFLINE
OFFLINE → (收到新数据) → ONLINE
ANY → (手动删除/解绑) → DELETED (软删除)
```

**离线检测策略**：
- 平台实时通知优先（Webhook/MQTT topo）
- 兜底：5s 定时扫描 Redis 心跳，60s 无数据 → 标记 OFFLINE + 告警
